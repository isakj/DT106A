\begin{frame}{Analysis of overfitting and data context}
  \textbf{Baseline (Nano)}
  \begin{itemize}
    \item Small model + modest data $\rightarrow$ learns gesture cues quickly; validation mAP@0.5 $\approx$ \textbf{0.98}.
    \item Loss curves and mAP curves in Colab align: enough \textbf{learning time} without obvious overfitting on this set.
  \end{itemize}
  \textbf{Stage 3 (larger model + heavy aug)}
  \begin{itemize}
    \item Validation mAP@0.5 \textbf{dropped} to \textbf{\StageThreeMap} vs.\ 0.98 baseline --- more capacity did not help.
    \item \textbf{Why metrics worsened:} augmentations broke \textbf{data context}. Flipping hands upside down or extreme rotation creates poses never labeled in the dataset; the model wastes capacity fitting synthetic views instead of real webcam gestures.
    \item Larger YOLO needs more diverse \emph{real} data or \emph{gentle} aug (brightness, slight rotation), not extreme inversions.
  \end{itemize}
  \textbf{Lesson:} match \textbf{model complexity} and \textbf{augmentation strength} to dataset size and how objects actually appear at inference.
\end{frame}

\begin{frame}{Key takeaways}
  \begin{itemize}
    \item \textbf{Cloud resources} make iteration fast (GPU, long training, saved plots in the notebook). Edge deployment proves the artifact is usable outside Colab.
    \item \textbf{Baseline Nano} reached \textbf{mAP@0.5 $\approx$ 0.98} on this small gesture set --- a simple model matched the problem size.
    \item \textbf{Over-engineering trap:} a bigger model plus aggressive augmentation \textbf{hurt} metrics; context-aware aug beats ``more aug.''
    \item \textbf{Deployment:} training metrics are not enough --- live timing, threshold, and stable class presence (150\,ms / 400\,ms rules) decide game behavior.
  \end{itemize}
  \vspace{0.35em}
  \textbf{Optional extensions}
  \begin{itemize}
    \item \texttt{tune()} hyperparameter search (notebook Step~6)
    \item Custom mild augmentation policy that improves over baseline
  \end{itemize}
